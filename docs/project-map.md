# MirrorTale Project Map

MirrorTale currently uses two separate GitHub repos. Keep them separate until
there is a clear reason to pay the migration cost of a monorepo.

## Repos

### `mirrortale-landing`

- Public Firebase Hosting site for `mirrortale.com`.
- Owns static HTML, CSS, browser JavaScript, public visual assets, analytics,
  legal/support pages, and local marketing automation tools.
- Must not contain AI keys, Stripe secrets, Firebase Admin credentials, service
  account files, or private order data.

### `mirrortale-engine`

- Cloud Run-ready API, admin dashboard, private Studio, checkout, webhooks,
  generation workers, and book asset generation.
- Owns all secret-bearing work: Stripe, Firebase Admin, Cloud Storage,
  Firestore, Resend, Gemini, Lulu, and Google Drive integrations.
- In local development it may serve the Studio UI, but production `GET /`
  remains a backend health response.

## Runtime Contract

- The landing page reads `<meta name="mirrortale-api-base-url">` in
  `index.html`.
- `script.js` builds the public `/orders` and `/contact` API URLs from that
  base URL.
- The production value should point at the engine Cloud Run API, usually
  `https://api.mirrortale.com`.
- If the base URL is blank, the landing forms keep their local mock behavior.

## Marketing Bridge

- `marketing-automation/` stays in this repo because it uses public brand,
  product, and landing-page assets.
- Production-faithful cover references are generated by the sibling engine repo
  through `marketing-automation/scripts/cover_library.py`.
- That bridge defaults to `../mirrortale-engine`, runs `npm run covers` there,
  then imports approved covers into
  `marketing-automation/references/book-covers/`.
- Generated social packages stay local outside the repo in the sibling
  `marketing-output/` folder. Set `MIRRORTALE_MARKETING_OUTPUT` to override
  that location.

## Folder Ownership

- `assets/` contains public, deployable site media.
- `docs/` contains deployment notes, backend contracts, and this map.
- `marketing-automation/references/` is the stable local drop zone for private
  reference images; image files remain ignored by git.
- `pomelli-upload-pack/` is the tracked brand handoff pack.
- `snapshots/` contains historical restore checkpoints from earlier design
  passes. It is not active app source.
